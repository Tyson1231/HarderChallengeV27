# Hunter Pathing Phase 2

This integration adds the first Hunter-only Baritone-inspired core:

- bounded A* planning
- weighted movement costs
- walking, one-block jumps and safe drops
- normal client key input execution
- smooth yaw control
- moving-target replanning
- stuck detection and recovery
- visible-target and last-seen goals
- a simplified Hunter process manager

Planned follow-on frameworks:

1. obstacle mining and tool selection
2. emergency placement, bridging and towering
3. builder plans and schematic execution
4. Elytra route planning, takeoff, rockets and landing
5. combat and inventory strategy

Only the Hunter client should run this control system.
